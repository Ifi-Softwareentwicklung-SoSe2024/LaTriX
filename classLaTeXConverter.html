Ausführliche Beschreibung

Die LaTeXConverter-Klasse. Enthält Methoden zur Konversion von MathNet.Numerics.LinearAlgebra-Matrizen und -Vektoren in einfügbaren LaTeX-Code.

Im LaTeX-Dokument werden keine weiteren Bibliotheken zur Matrizen- und Vektoranzeige benötigt, der Code muss lediglich im Mathematikmodus eingefügt werden.

inlinestatic

Konvertiert die übergebene Matrix matrix in einen LaTeX-Code, der diese anzeigen kann.

Template-Parameter
TEin Datentyp, aus dem eine MathNet.Numerics.LinearAlgebra.Matrix gebaut werden kann (Double, Single, Complex, Complex32).
Parameter
matrixDie zu konvertierende Matrix.
alignmentAnordnung der Matrixelemente. Eines von LaTeXConverter.Alignment.center (zentriert), LaTeXConverter.Alignment.left (linksbündig) oder LaTeXConverter.Alignment.right (rechtsbündig). Standardmäßig LaTeXConverter.Alignment.center.
Rückgabe
Ein LaTeX-Code, der direkt in ein LaTeX-Dokument (im Mathematikmodus!) eingefügt werden kann, um die konvertierte Matrix anzuzeigen.

Der erzeugte LaTeX-Code besitzt keine eigene Kontrollsequenz zur Einleitung des Mathematikmodus. Der Code muss explizit im Mathematikmodus eingefügt werden.

Aus Kompatibilitätsgründen (und da LaTeX diese ebenfalls standardmäßig als Dezimaltrennzeichen nutzt) wird als Trennzeichen zwischen "Vorkommastellen" und "Nachkommastellen" der im englischen Sprachraum übliche Punkt genutzt.

Der erzeugte Code arbeitet mit der LaTeX-Kontrollsequenz \begin{array} statt \begin{pmatrix}, um auch ohne den Import des amsmath-Moduls zu funktionieren.
